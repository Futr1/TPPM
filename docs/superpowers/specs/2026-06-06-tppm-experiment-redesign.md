# TPPM 实验结构重设计方案

**日期**：2026-06-06
**状态**：待实施
**范围**：仅修改实验部分（第4-8节），不修改方法部分（第3节）

---

## 1. 设计目标

基于现有三个数据集（PsyDial、LoCoMo、PersonaMem），围绕 TPPM 的核心创新点重新设计实验结构，使每个创新点都有明确的验证路径。

**核心变化**：
- 移除 TalkLife MoC（实验二 Layer 2）
- 不引入新数据集
- 通过消融实验升级和分析子节升级来验证动态机制（衰减/固化）
- 场景条件化分支不单独验证，保留为常规消融项
- 混合式设计由扰码/噪声记忆对照覆盖，不需要额外实验

## 2. TPPM 核心创新点与验证路径映射

| 创新点 | 核心机制 | 验证路径 |
|--------|---------|---------|
| ①心理画像中心记忆 | PPMU 7维槽位，只保留心理相关信息 | PersonaMem Acc + Unsupported Rate + 消融（w/o Structured Slots） |

## 4. 各实验详细设计

### 4.1 实验一：基本不变

与现版完全一致：
- Layer 1：PsyDial 101段测试对话，8维咨询技能 + OA，GPT-4o 评分
- Layer 2：LoCoMo QA（5类F1）+ Event Summarization（FactScore + ROUGE-L）
- 两层综合解读保留

**唯一微调**：综合解读段落中，将"心理领域特异性由实验二 Layer 2（TalkLife MoC）独立验证"改为"动态记忆机制的有效性由实验二 Layer 2 独立验证"。

### 4.2 实验二：重构

#### 4.2.1 标题与定位

**标题**：实验二：心理画像记忆建模与动态机制验证

**实验目的**：实验一的双层评估验证了 TPPM 在回复生成中的效果及其与通用记忆能力的关联。实验二进一步验证：(1) 系统是否准确抽取了用户画像槽位（Layer 1·PersonaMem）；(2) TPPM 的三个动态记忆机制——状态→特质固化、类型条件衰减、场景条件分支——是否各自不可替代（Layer 2·机制消融验证）。

#### 4.2.2 Layer 1：PersonaMem（不变）

与现版完全一致。包括：
- 评估协议、设计优势、局限性说明
- Table: PersonaMem 画像槽位抽取评估（6列）
- 论证逻辑

#### 4.2.3 Layer 2：动态记忆机制消融验证（新）

**设计动机**。Layer 1 验证了画像槽位的静态抽取质量，但 TPPM 的核心创新在于动态记忆管理——时间衰减、状态→特质固化、场景条件分支。Layer 2 通过消融实验验证每个动态机制的独立贡献，回答"这些机制是否各自不可替代"。

**与实验三消融的区别**。实验三的五模块全局消融在全部关键指标上报告退化，目的是评估各模块对最终性能的总体贡献。Layer 2 的机制消融聚焦于三个动态机制，针对每个机制设计最敏感的诊断指标和最直接的验证条件，目的是提供机制有效性的因果证据。

##### 2a. 固化机制消融验证

**消融条件**：关闭公式(11)的固化门控——所有瞬时画像记忆在会话结束后直接丢弃，不再积累为长期特质。

**主指标**：
- PersonaMem Q4（偏好演化追踪）：固化是跨session积累的前提，移除后Q4退化应最大
- LoCoMo QA Multi-hop F1：跨session推理依赖长期记忆，移除后退化

**辅助验证**：
- K敏感性分析：在K∈{1,2,3,5,7,10}范围内扫描，报告Trait Consolidation Precision/Recall（而非合并F1），展示K的Precision-Recall权衡
- 高不稳定压力测试中的固化抑制率

**对照基线**：Long-Context / Summary-Memory / RAG-Dense 在相同指标上作为参照——若 TPPM（完整）显著优于这些基线且消融后退化至基线水平，则说明动态机制正是 TPPM 相对这些替代方案的优势来源。

**论证逻辑**：若Q4退化最大且K敏感性显示清晰的Precision-Recall权衡，则固化机制不可替代——它既防止了一次性波动被误固化（高Precision），又确保了稳定特质被及时积累（高Recall）。

##### 2b. 衰减机制消融验证

**消融条件**：关闭公式(13)的类型条件衰减——所有画像属性值在时间推移中保持不变（λ_d≡0）。

**主指标**：
- LoCoMo QA Temporal F1：时间推理最敏感于陈旧信息干扰
- PersonaMem Q3（识别最新偏好）：衰减机制确保陈旧偏好被弱化，使最新偏好凸显

**辅助验证——差分衰减对比**：
在LoCoMo上比较三种衰减策略：
1. 无衰减（所有s_i固定）
2. 统一衰减（单一λ）
3. TPPM差分衰减（λ_affect > λ_stressor > λ_coping ≈ λ_support > λ_trait）

报告Temporal F1和PersonaMem Q3在三种策略下的表现。预期差分衰减显著优于统一衰减。

**对照基线**：Long-Context / Summary-Memory / RAG-Dense 在相同指标上作为参照。

**论证逻辑**：若(1)移除衰减导致Temporal F1和Q3退化，(2)差分衰减优于统一衰减，则类型条件衰减的设计不可替代——不仅是"需要衰减"，更是"需要不同维度不同衰减率"。

##### 2c. 场景分支机制消融验证

**消融条件**：将公式(3)的场景条件分支替换为无条件的全局画像更新。

**主指标**：
- PersonaMem Q7（跨场景泛化）：场景分支使系统能在不同情境下读取不同的画像分支，移除后泛化能力退化
- LoCoMo QA Adversarial F1：对抗性问题需要区分不同说话人/情境，无条件更新导致错误归属

**辅助验证**：矛盾信号压力测试中的False Conflict Rate。

**对照基线**：Long-Context / Summary-Memory / RAG-Dense 在相同指标上作为参照。

**论证逻辑**：若Q7和Adversarial F1退化，且矛盾信号测试中False Conflict Rate上升，则场景分支不可替代——它使系统能区分"情境差异"与"真实心理转折"。

#### 4.2.4 Layer 2 论证逻辑总结

Layer 2 的三个消融实验构成收敛验证：

```
固化消融 → Q4退化 + K权衡 → 固化不可替代
衰减消融 → Temporal F1退化 + 差分>统一 → 类型条件衰减不可替代
分支消融 → Q7退化 + False Conflict↑ → 场景分支不可替代
```

与 Layer 1（画像更准）和实验一（回复更好+记忆更好）组合，形成完整的论证链：
- 实验一：TPPM整体更好
- 实验二 Layer 1：更好是因为画像更准
- 实验二 Layer 2：画像更准是因为每个动态机制各司其职

## 7. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 无独立数据集验证动态机制 | 审稿人可能质疑"只有消融不够" | 消融 + 敏感性分析 + 压力测试三线收敛；且消融实验在三个不同数据集上交叉验证 |
| PersonaMem Q4/Q3/Q7作为消融诊断指标可能退化幅度不大 | 论证力度弱 | 如果退化不显著，则在讨论中诚实报告，并分析原因（如therapy子集session数不足） |
| 去掉TalkLife MoC后实验二偏轻 | 说服力下降 | Layer 2 三个子实验 + K敏感性 + 差分衰减 + 会话数敏感性，总分析量不减 |
| 消融表格列数减少 | 信息量减少 | 可在PersonaMem Acc列旁拆出Q3/Q4/Q7子指标列，增加诊断粒度 |
